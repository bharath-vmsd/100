The application follows a component-based architecture with clear separation of concerns and modular design patterns.

Component Structure

App.js

Root component managing application routing and global layout structure.

NavBar.js

Persistent navigation component rendered across all routes for consistent UX.

HomePage.js

Landing page component serving as the entry point for visitors.

AboutPage.js

Informational page providing blog context and author information.

ArticleListPage.js

Index page displaying all available articles with navigation capabilities.

ArticlePage.js

Detailed view component for rendering individual article content.

NotFoundPage.js

Fallback component for handling 404 errors and invalid routes.

üíé Code Practices

  • Component-Based Architecture: Modular design promoting code reusability and maintainability through isolated, self-contained components.
  • Organized File Structure: Logical separation with dedicated directories for pages (src/pages/) and reusable components (src/components/).
  • CSS Modules: Scoped styling preventing class name collisions and ensuring style encapsulation at the component level.
  • React Router DOM: Declarative routing configuration with dynamic parameter handling and nested route support.
